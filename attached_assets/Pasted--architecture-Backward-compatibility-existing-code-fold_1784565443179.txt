আগের architecture পরিবর্তন করবে না। Backward compatibility বজায় রাখবে। কোনো existing code, folder structure বা configuration অপ্রয়োজনীয়ভাবে পরিবর্তন, rename বা refactor করবে না। কোনো feature implement করবে না যা এই phase-এর scope-এর বাইরে।

## Step 0 — Repository Analysis (অবশ্যই প্রথমে করবে)

কোনো code লেখার আগে:

1. পুরো repository scan করবে।
2. README.md সম্পূর্ণ পড়বে।
3. Project structure বিশ্লেষণ করবে।
4. বর্তমান architecture বুঝবে।
5. সর্বশেষ commit পর্যন্ত কী কী implement হয়েছে তার summary তৈরি করবে।
6. Completed phase এবং pending phase শনাক্ত করবে।
7. যদি কোনো requirement অস্পষ্ট হয় তাহলে code লেখা শুরু করার আগে প্রশ্ন করবে।
